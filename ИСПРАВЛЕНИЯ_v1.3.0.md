# 🔧 Исправления в версии 1.3.0

## 🎯 Главное исправление: РАДИКАЛЬНОЕ РЕШЕНИЕ проблемы с Clowd.Squirrel.dll

### ❌ Проблема:
Во всех предыдущих версиях (1.2.7, 1.2.8, 1.2.9) при запуске установленного приложения возникала ошибка:
```
Could not load file or assembly 'Clowd.Squirrel, Culture=neutral, PublicKeyToken=null'
Не удается найти указанный файл.
```

**Причина**: `dotnet publish` НЕ копирует `Clowd.Squirrel.dll` в папку публикации, потому что библиотека используется только через рефлексию (Assembly.Load), и компилятор считает её "неиспользуемой".

### ✅ Решение в v1.3.0:
Добавлен новый шаг в GitHub Actions workflow: **принудительное копирование DLL из NuGet кэша**

#### Изменения в `.github/workflows/build-and-release.yml`:

```yaml
- name: Copy Clowd.Squirrel.dll manually
  run: |
    # Находим Clowd.Squirrel.dll в NuGet кэше
    $squirrelDll = Get-ChildItem -Path "$env:USERPROFILE\.nuget\packages\clowd.squirrel\2.11.1\lib\net8.0\Clowd.Squirrel.dll" -ErrorAction SilentlyContinue
    if ($squirrelDll) {
      Write-Host "✅ Найден Clowd.Squirrel.dll: $($squirrelDll.FullName)" -ForegroundColor Green
      Copy-Item $squirrelDll.FullName -Destination "bin/Release/net8.0-windows/win-x64/publish/" -Force
      Write-Host "✅ Clowd.Squirrel.dll скопирован в publish папку" -ForegroundColor Green
    } else {
      Write-Host "❌ Clowd.Squirrel.dll НЕ найден в NuGet кэше!" -ForegroundColor Red
      exit 1
    }
    
    # Проверяем что файл скопирован
    if (Test-Path "bin/Release/net8.0-windows/win-x64/publish/Clowd.Squirrel.dll") {
      Write-Host "✅ ПОДТВЕРЖДЕНО: Clowd.Squirrel.dll присутствует в publish папке" -ForegroundColor Green
    } else {
      Write-Host "❌ ОШИБКА: Clowd.Squirrel.dll НЕ скопирован!" -ForegroundColor Red
      exit 1
    }
  shell: pwsh
```

#### Как это работает:
1. После `dotnet publish` ищем `Clowd.Squirrel.dll` в NuGet кэше GitHub Actions runner
2. Принудительно копируем DLL в папку `bin/Release/net8.0-windows/win-x64/publish/`
3. Проверяем что файл успешно скопирован (если нет - workflow упадёт с ошибкой)
4. Squirrel.exe упакует все файлы из publish папки, включая вручную скопированную DLL
5. Установщик включит `Clowd.Squirrel.dll` в установленную версию приложения

### 🎯 Ожидаемый результат:
После установки v1.3.0:

## 🔍 Технические детали:

### Почему dotnet publish не копирует DLL:
В `Services/AutoUpdateService.cs` используется рефлексия для загрузки Clowd.Squirrel:
```csharp
var squirrelAssembly = Assembly.Load("Clowd.Squirrel");
```

Компилятор не видит прямых ссылок на типы из Clowd.Squirrel (нет `using Clowd.Squirrel;`), поэтому считает библиотеку неиспользуемой и не включает в output.

### Почему используется рефлексия:
Прямой импорт `using Clowd.Squirrel;` вызывает ошибки компиляции в WPF temporary projects:
```
error CS0234: The type or namespace name 'Squirrel' does not exist in the namespace 'Clowd'
```
