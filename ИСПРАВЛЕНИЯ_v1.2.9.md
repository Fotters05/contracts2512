# 🔧 Исправления в версии 1.2.9

## Основная проблема:
**Clowd.Squirrel.dll отсутствует в установленной версии приложения**

### Симптомы:
- При запуске установленного приложения появляется ошибка:
  ```
  Could not load file or assembly 'Clowd.Squirrel, Culture=neutral, PublicKeyToken=null'
  Не удается найти указанный файл.
  ```
- Файл `Clowd.Squirrel.dll` отсутствует в папке `C:\Users\[USER]\AppData\Local\Contract2512\app-X.X.X\`

### История попыток исправления:
1. **v1.2.7**: Добавили `<CopyLocalLockFileAssemblies>true</CopyLocalLockFileAssemblies>` в Contract2512.csproj
   - Результат: ❌ НЕ СРАБОТАЛО (DLL всё ещё отсутствует)
   
2. **v1.2.8**: Изменили workflow на `--self-contained true`
   - Результат: ❌ НЕ СРАБОТАЛО (DLL всё ещё отсутствует)
   - Причина: `--self-contained true` включает .NET Runtime, но не гарантирует включение NuGet пакетов

3. **v1.2.9**: Используем `-p:CopyLocalLockFileAssemblies=true` как параметр dotnet publish
   - Ожидаемый результат: ✅ ДОЛЖНО СРАБОТАТЬ

## Изменения в v1.2.9:

### 1. Изменён `.github/workflows/build-and-release.yml`
```yaml
# БЫЛО (v1.2.8):
- name: Build and Publish
  run: dotnet publish Contract2512.csproj -c Release -r win-x64 --self-contained true -p:PublishSingleFile=false

# СТАЛО (v1.2.9):
- name: Build and Publish
  run: dotnet publish Contract2512.csproj -c Release -r win-x64 --self-contained false -p:PublishSingleFile=false -p:CopyLocalLockFileAssemblies=true
```

### Почему это должно сработать:
- `-p:CopyLocalLockFileAssemblies=true` как параметр publish команды **принудительно** копирует все зависимости из project.assets.json
- Это включает все NuGet пакеты, включая Clowd.Squirrel.dll
- Параметр в csproj работает для обычной сборки, но может игнорироваться при publish
- Параметр в команде publish имеет более высокий приоритет

5. Убедиться что файл `Clowd.Squirrel.dll` **присутствует**
