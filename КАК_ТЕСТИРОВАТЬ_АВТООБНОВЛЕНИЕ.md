# 🧪 Как правильно тестировать автообновление

## ⚠️ КРИТИЧЕСКИ ВАЖНО!

**Squirrel НЕ РАБОТАЕТ в Debug режиме из Visual Studio!**

Если ты запускаешь `bin/Debug/net8.0-windows/Contract2512.exe` - автообновление **НЕ БУДЕТ РАБОТАТЬ**.

## ✅ Правильный способ тестирования

### Шаг 1: Установи приложение через Setup.exe

1. Открой GitHub Releases: https://github.com/Fotters05/contracts2512/releases
2. Скачай **последний** релиз (например v1.1.7)
3. Скачай файл `Contract2512Setup.exe`
4. **Запусти Setup.exe** - он установит приложение в:
   ```
   C:\Users\rhali\AppData\Local\Contract2512\
   ```

### Шаг 2: Запусти УСТАНОВЛЕННУЮ версию

**НЕ ЗАПУСКАЙ** из Visual Studio!
**НЕ ЗАПУСКАЙ** из `bin/Debug/`!

Запусти одним из способов:
- Из меню Пуск → "Contract2512"
- Из папки: `C:\Users\rhali\AppData\Local\Contract2512\Contract2512.exe`

### Шаг 3: Создай новую версию

1. Измени версию в `Contract2512.csproj`:
   ```xml
   <Version>1.2.0</Version>
   ```

2. Закоммить и создай тег:
   ```powershell
   git add Contract2512.csproj
   git commit -m "Bump version to 1.2.0"
   git tag v1.2.0
   git push origin main
   git push origin v1.2.0
   ```

3. Подожди 5-10 минут пока GitHub Actions создаст релиз

### Шаг 4: Проверь обновление

1. Запусти **УСТАНОВЛЕННУЮ** версию (из Пуска или из AppData)
2. При запуске приложение автоматически проверит обновления
3. Если обновление найдено - появится окно с предложением обновиться

## 🔍 Как посмотреть логи

Логи Squirrel пишутся в:
```
C:\Users\rhali\AppData\Local\Contract2512\SquirrelSetup.log
```

Открой этот файл чтобы увидеть что происходит при проверке обновлений.

## ❌ Почему Debug не работает?

Squirrel требует:
1. Приложение должно быть **установлено** через Setup.exe
2. Приложение должно находиться в папке `AppData\Local\{AppName}\`
3. Должны быть файлы `.nupkg` и `RELEASES` в папке приложения

В Debug режиме всего этого нет, поэтому Squirrel просто не загружается.
