# Исправления в версии 1.2.2

## Проблема
При попытке собрать версию 1.2.1 на GitHub Actions возникала ошибка компиляции:
```
CS0246: Не удалось найти тип или имя пространства имен "Clowd"
```

## Причина
В файле `Services/AutoUpdateService.cs` была строка:
```csharp
using Clowd.Squirrel;
```

Эта строка вызывает проблемы с временными WPF проектами, которые создаются во время компиляции.

## Решение
Заменили прямой импорт на использование рефлексии:

### Было:
```csharp
using Clowd.Squirrel;

public class AutoUpdateService
{
    private UpdateManager _updateManager;
    // ...
}
```

### Стало:
```csharp
using System.Reflection;

public class AutoUpdateService
{
    public async Task<UpdateInfo> CheckForUpdatesAsync()
    {
        // Загружаем Squirrel через рефлексию
        var squirrelAssembly = Assembly.Load("Clowd.Squirrel");
        var updateManagerType = squirrelAssembly.GetType("Clowd.Squirrel.UpdateManager");
        
        // Создаём UpdateManager через рефлексию
        var mgr = Activator.CreateInstance(updateManagerType, _updateUrl);
        
        // Вызываем методы через рефлексию
        var checkMethod = updateManagerType.GetMethod("CheckForUpdate");
        var updateTask = checkMethod?.Invoke(mgr, null) as Task<object>;
        // ...
    }
}
```

## Результат
- ✅ Код компилируется без ошибок
- ✅ WPF временные проекты не вызывают проблем
- ✅ Автообновление работает через рефлексию
- ✅ GitHub Actions может собрать релиз

## Файлы изменены
- `Services/AutoUpdateService.cs` - использует рефлексию вместо прямого импорта

- `App.xaml.cs` - уже использовал рефлексию (не изменялся)
