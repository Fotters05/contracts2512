# 🚀 Руководство по автообновлению Contract2512 через Clowd.Squirrel

## Что такое Clowd.Squirrel?

**Clowd.Squirrel** — это современная версия Squirrel.Windows, профессиональный фреймворк для установки и автоматического обновления Windows-приложений. Он используется в таких проектах как GitHub Desktop, Slack, Discord.

### Преимущества Clowd.Squirrel:

✅ **Установщик Setup.exe** — пользователи устанавливают приложение как обычную программу  
✅ **Автоматические обновления** — приложение обновляется в фоне без участия пользователя  
✅ **Ярлыки на рабочем столе** — создаются автоматически при установке  
✅ **Удаление через "Программы и компоненты"** — стандартная деинсталляция Windows  
✅ **Delta-обновления** — скачиваются только изменения, а не весь .exe  
✅ **Поддержка .NET 8.0** — работает с современными версиями .NET

---

## 🎯 Как это работает

### Для разработчика:

1. **Обновляешь версию** в `Contract2512.csproj`
2. **Создаешь git tag** → `git tag v1.0.1 && git push origin v1.0.1`
3. **GitHub Actions автоматически:**
   - Собирает проект
   - Создает Squirrel-пакеты (Setup.exe, RELEASES, .nupkg)
   - Публикует Release на GitHub

### Для пользователя:

**Первая установка:**
1. Скачивает `Setup.exe` из GitHub Releases
2. Запускает установщик
3. Приложение устанавливается в `%LocalAppData%\Contract2512`
4. Создается ярлык на рабочем столе

**Автообновление:**
1. Запускает приложение
2. Clowd.Squirrel проверяет обновления в фоне
3. Если есть новая версия → показывается окно обновления
4. Пользователь нажимает "Обновить сейчас"
5. Обновление скачивается и устанавливается
6. Приложение перезапускается автоматически

## 📝 Процесс выпуска новой версии

### 1. Обнови версию в `Contract2512.csproj`

```xml
<PropertyGroup>
  <Version>1.0.1</Version>  <!-- Увеличь версию -->
  <AssemblyVersion>1.0.1.0</AssemblyVersion>
  <FileVersion>1.0.1.0</FileVersion>
</PropertyGroup>
```

### 2. Сделай commit и push

```bash
git add .
git commit -m "Release v1.0.1: Добавлены новые функции"
git push origin main
```

### 3. Создай тег версии

```bash
# Создать тег
git tag v1.0.1

# Отправить тег на GitHub
git push origin v1.0.1
```

### 4. GitHub Actions автоматически создаст:

- ✅ `Setup.exe` — установщик для новых пользователей
- ✅ `RELEASES` — файл с информацией о версиях
- ✅ `Contract2512-1.0.1-full.nupkg` — полный пакет обновления
- ✅ `Contract2512-1.0.1-delta.nupkg` — дельта-обновление (только изменения)

### 5. Пользователи получат обновление!

- При запуске приложения Clowd.Squirrel проверит обновления
- Появится окно с предложением обновиться
- Обновление скачается и установится автоматически

## ⚙️ Настройка репозитория (один раз)

### 1. Создай репозиторий на GitHub

```bash
# Инициализируй git (если еще не сделал)
git init
git add .
git commit -m "Initial commit"

# Добавь remote
git remote add origin https://github.com/твой-username/contracts2512.git

# Отправь код
git push -u origin main
```

### 2. Настрой GitHub Actions

Файл `.github/workflows/build-and-release.yml` уже создан и настроен!

Он автоматически:
- Собирает проект при создании тега
- Создает Clowd.Squirrel-пакеты
- Публикует Release с Setup.exe

**Ничего дополнительно настраивать не нужно!**

### 3. Добавь настройки в `.env` (опционально)

Если твой репозиторий **приватный**, добавь в `.env`:

```env
# GitHub Settings (для приватных репозиториев)
GITHUB_OWNER=твой-username
GITHUB_REPO=contracts2512
GITHUB_TOKEN=ghp_xxxxxxxxxxxxx  # Personal Access Token с правами repo
```

Для **публичных** репозиториев токен не нужен!

## 🔍 Что создает GitHub Actions

После push тега в Release будут загружены:

### 1. `Setup.exe` (установщик)
- Для новых пользователей
- Устанавливает приложение в `%LocalAppData%\Contract2512`
- Создает ярлык на рабочем столе
- Регистрирует в "Программы и компоненты"

### 2. `RELEASES` (файл версий)
- Содержит список всех версий
- Используется Squirrel для проверки обновлений

### 3. `Contract2512-1.0.1-full.nupkg` (полный пакет)
- Содержит все файлы приложения
- Используется для первой установки

### 4. `Contract2512-1.0.1-delta.nupkg` (дельта-пакет)
- Содержит только изменения с предыдущей версии
- Экономит трафик при обновлении

## 🔐 Настройка для приватного репозитория

Если твой репозиторий **приватный**, нужен GitHub Personal Access Token.

### 1. Создай токен

1. Перейди на https://github.com/settings/tokens
2. Нажми **"Generate new token (classic)"**
3. Дай название: `Contract2512 Auto-Update`
4. Срок действия: **No expiration**
5. Выбери scope: **✅ repo** (Full control of private repositories)
6. Нажми **"Generate token"**
7. **ВАЖНО:** Скопируй токен сразу!

### 2. Добавь токен в `.env`

```env
# GitHub Settings
GITHUB_OWNER=твой-username
GITHUB_REPO=contracts2512
GITHUB_TOKEN=ghp_xxxxxxxxxxxxx
```

### 3. Готово!

Теперь приложение может скачивать обновления из приватного репозитория.

## 🔒 Безопасность токена

### ⚠️ ВАЖНО: Не коммить токен в Git!

Файл `.env` уже добавлен в `.gitignore`, поэтому токен не попадет в репозиторий.

### Что делать, если токен утек?

1. Перейди на https://github.com/settings/tokens
2. Найди свой токен
3. Нажми **"Delete"**
4. Создай новый токен
5. Обнови `.env` с новым токеном

## ❓ Частые вопросы

### Где устанавливается приложение?

```
%LocalAppData%\Contract2512\
```

Это стандартное место для приложений Windows (например: `C:\Users\Username\AppData\Local\Contract2512\`)

### Как удалить приложение?

1. **Через "Программы и компоненты":**
   - Панель управления → Программы → Удалить программу
   - Найди "Contract2512" → Удалить

2. **Вручную:**
   - Удали папку `%LocalAppData%\Contract2512`
   - Удали ярлык с рабочего стола

### Приложение не видит обновления?

Проверь:
1. ✅ Release создан на GitHub
2. ✅ К Release прикреплены файлы `Setup.exe`, `RELEASES`, `.nupkg`
3. ✅ Версия в GitHub новее, чем в приложении
4. ✅ Если репозиторий приватный — токен добавлен в `.env`

### Как отменить обновление?

Пользователь может нажать "Позже" в окне обновления.

### Как сделать обязательное обновление?

Убери кнопку "Позже" в `UpdateWindow.xaml`:
```xml
<!-- Закомментируй или удали -->
<!-- <Button x:Name="LaterButton" ... /> -->
```

### Как изменить описание в Release?

Отредактируй файл `.github/workflows/build-and-release.yml`:
```yaml
body: |
  ## 🎉 Новая версия v${{ steps.get_version.outputs.VERSION }}
  
  ### Что нового:
  - Твои изменения
  - Исправления
```

### Можно ли обновить приложение вручную?

Да! Просто скачай новый `Setup.exe` и запусти. Он обновит существующую установку.

### Как работают дельта-обновления?

Clowd.Squirrel сравнивает текущую версию с новой и скачивает только изменившиеся файлы. Это экономит трафик и ускоряет обновление.
