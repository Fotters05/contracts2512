# 🚀 Руководство по автообновлению Contract2512

## Как это работает

1. **Ты делаешь изменения** → commit → push в GitHub
2. **Создаешь тег версии** → GitHub Actions автоматически собирает .exe
3. **Создается Release** → пользователи получают уведомление об обновлении
4. **Приложение обновляется** → автоматически скачивает и устанавливает новую версию

---

## 🔐 Настройка для приватного репозитория (один раз)

### 1. Создай GitHub Personal Access Token

1. Перейди на https://github.com/settings/tokens
2. Нажми **"Generate new token (classic)"**
3. Дай название токену: `Contract2512 Auto-Update`
4. Выбери срок действия: **No expiration** (без срока действия)
5. Выбери scope: **✅ repo** (Full control of private repositories)
6. Нажми **"Generate token"**
7. **ВАЖНО:** Скопируй токен сразу! Он больше не будет показан

### 2. Добавь токен в `.env`

Открой файл `.env` в корне проекта и заполни:

```env
# GitHub Auto-Update Settings
GITHUB_OWNER=твой-username        # Твой GitHub username
GITHUB_REPO=contracts2512         # Название репозитория
GITHUB_TOKEN=ghp_xxxxxxxxxxxxx   # Токен, который ты скопировал
```

Пример:
```env
GITHUB_OWNER=rhali
GITHUB_REPO=contracts2512
GITHUB_TOKEN=ghp_1234567890abcdefghijklmnopqrstuvwxyz
```

### 3. Готово! Теперь приложение может обновляться из приватного репозитория

## 📝 Процесс выпуска новой версии

### 1. Обнови версию в `Contract2512.csproj`

```xml
<PropertyGroup>
  <Version>1.0.1</Version>  <!-- Увеличь версию -->
  <AssemblyVersion>1.0.1.0</AssemblyVersion>
  <FileVersion>1.0.1.0</FileVersion>
</PropertyGroup>
```

### 2. Сделай commit и push

```bash
git add .
git commit -m "Release v1.0.1: Добавлены новые функции"
git push origin main
```

### 3. Создай тег версии

```bash
# Создать тег
git tag v1.0.1

# Отправить тег на GitHub
git push origin v1.0.1
```

### 4. GitHub Actions автоматически:
- ✅ Соберет проект
- ✅ Создаст Release
- ✅ Прикрепит Contract2512.exe к релизу

### 5. Пользователи получат обновление!
- При запуске приложения появится окно с предложением обновиться
- Нажмут "Обновить сейчас" → приложение скачает и установит новую версию
- Приложение автоматически перезапустится

Он автоматически:
- Собирает проект при создании тега
- Создает Release
- Прикрепляет .exe файл

**Ничего дополнительно настраивать не нужно!**

## 🔒 Безопасность токена

### ⚠️ ВАЖНО: Не коммить токен в Git!

Файл `.env` уже добавлен в `.gitignore`, поэтому твой токен не попадет в репозиторий.

### Что делать, если токен утек?

1. Перейди на https://github.com/settings/tokens
2. Найди свой токен
3. Нажми **"Delete"**
4. Создай новый токен (см. инструкцию выше)
5. Обнови `.env` с новым токеном

### Как передать токен другим разработчикам?

1. Каждый разработчик создает свой токен
2. Каждый добавляет токен в свой локальный `.env`
3. **НЕ передавай токен через Git, email или мессенджеры!**

## ❓ Частые вопросы

### Приложение не видит обновления?

Проверь:
1. ✅ Правильно указан `GITHUB_OWNER` и `GITHUB_REPO` в `.env`
2. ✅ Токен `GITHUB_TOKEN` добавлен в `.env`
3. ✅ Токен имеет scope `repo`
4. ✅ Release создан на GitHub
5. ✅ К Release прикреплен файл `.exe`
6. ✅ Версия в GitHub новее, чем в приложении

### Ошибка "API rate limit exceeded"?

Это значит, что токен не используется или неправильный. Проверь:
- Токен добавлен в `.env`
- Токен начинается с `ghp_`
- Токен не истек

### Как отменить обновление?

Пользователь может нажать "Позже" в окне обновления.

### Как сделать обязательное обновление?

Убери кнопку "Позже" в `UpdateWindow.xaml`:
```xml
<!-- Закомментируй или удали -->
<!-- <Button x:Name="LaterButton" ... /> -->
```

### Как изменить описание в Release?

Отредактируй файл `.github/workflows/build-and-release.yml`:
```yaml
body: |
  ## Твое описание
  - Что нового
  - Исправления
```
