# 🔍 Почему не работало автообновление

## ❌ Проблема

Когда вы делали обычный `git push`, приложение **НЕ получало обновления**.

## 🎯 Причина

GitHub Actions настроен на запуск **ТОЛЬКО при создании тега**:

```yaml
on:
  push:
    tags:
      - 'v*.*.*'  # Запускается ТОЛЬКО на теги v1.0.0, v1.1.0 и т.д.
```

### Что происходило:

1. ✅ Вы делали изменения в коде
2. ✅ Делали `git add .` и `git commit`
3. ✅ Делали `git push origin main`
4. ❌ **GitHub Actions НЕ запускался** (нет тега!)
5. ❌ **Release НЕ создавался**
6. ❌ **Приложение НЕ находило обновлений**

### Что должно происходить:

1. ✅ Вы делаете изменения в коде
2. ✅ Обновляете версию в `Contract2512.csproj`
3. ✅ Делаете `git add .` и `git commit`
4. ✅ **Создаете тег:** `git tag v1.1.5`
5. ✅ **Пушите тег:** `git push origin v1.1.5`
6. ✅ **GitHub Actions запускается**
7. ✅ **Создается Release с Setup.exe и пакетами**
8. ✅ **Приложение находит обновление!**

---

## ✅ Решение

### Используйте скрипт `release.ps1`:

```powershell
# Создать новый релиз
.\release.ps1 -Version 1.1.5
```

Скрипт автоматически:
- Обновит версию в проекте
- Создаст commit
- Создаст тег
- Запушит на GitHub
- GitHub Actions создаст Release

## 🔍 Как проверить, что все работает

### 1. После создания тега проверьте GitHub Actions:
https://github.com/Fotters05/contracts2512/actions

Должен появиться процесс "Build and Release" (зеленая галочка через ~5-10 минут)

### 2. Проверьте Release:
https://github.com/Fotters05/contracts2512/releases

Должен появиться новый Release с файлами:
- ✅ `Setup.exe`
- ✅ `RELEASES`
- ✅ `Contract2512-1.1.5-full.nupkg`
- ✅ `Contract2512-1.1.5-delta.nupkg`

### 3. Запустите старую версию приложения:

Должно появиться окно обновления с кнопкой "Обновить сейчас"!

## 💡 Важные моменты

### ⚠️ Обычный push НЕ создает обновление!

```bash
git push origin main  # ❌ GitHub Actions НЕ запустится
```

### ✅ Только push тега создает обновление!

```bash
git push origin v1.1.5  # ✅ GitHub Actions запустится
```

### 📝 Версия в проекте должна совпадать с тегом

```xml
<!-- Contract2512.csproj -->
<Version>1.1.5</Version>  <!-- Должна совпадать с тегом v1.1.5 -->
```

### 🔄 Приложение проверяет обновления при запуске

Проверка происходит в `App.xaml.cs` → `CheckForUpdatesAsync()`
