## Проблема 1: Парсер не находит папку parser_nodejs ❌ → ✅

### Что было:
- Парсер использовал жестко заданный относительный путь `@"..\..\..\"`
- Это работало только в режиме Debug во время разработки
- После публикации приложения структура папок менялась и парсер не мог найти папку
- Ошибка: `Папка не найдена: C:\Users\rhali\AppData\parser_nodejs`

### Что исправлено:

#### 1. `Views/ParserWindow.xaml.cs`
- ✅ Добавлен метод `GetParserPath()` который:
  - Сначала проверяет путь для опубликованного приложения (parser_nodejs рядом с exe)
  - Если не найдено, использует путь для режима разработки
  - Возвращает правильный путь в зависимости от окружения

#### 2. `Services/NodePackageService.cs`
- ✅ Обновлена логика определения пути к парсеру
- ✅ Теперь использует ту же логику что и ParserWindow
- ✅ Работает как в Debug, так и в опубликованной версии

### Как это работает:
```csharp
// 1. Проверяем опубликованную версию (parser_nodejs рядом с exe)
string publishedPath = Path.Combine(exeDir, "parser_nodejs");
if (Directory.Exists(publishedPath))
    return publishedPath;

// 2. Если не найдено, используем путь разработки
string projectRoot = Path.GetFullPath(Path.Combine(exeDir, @"..\..\..\"));
return Path.Combine(projectRoot, "parser_nodejs");
```

## Проблема 2: Автообновление не работает ❌ → ✅

### Что было:
- `AutoUpdateService.cs` использовал рефлексию для загрузки Squirrel
- Это было сделано чтобы избежать проблем с WPF временными проектами
- Но пакет `Clowd.Squirrel` уже был добавлен в проект!
- Рефлексия усложняла код и могла приводить к ошибкам

### Что исправлено:

#### 1. `Services/AutoUpdateService.cs`
- ✅ Удалена вся логика с рефлексией
- ✅ Добавлен прямой импорт `using Clowd.Squirrel;`
- ✅ Используется прямой API Squirrel вместо рефлексии
- ✅ Добавлено подробное логирование всех операций
- ✅ Улучшена обработка ошибок

#### 2. `App.xaml.cs`
- ✅ Раскомментирован и исправлен метод `HandleSquirrelEventsAsync()`
- ✅ Добавлен импорт `using Clowd.Squirrel;`
- ✅ Теперь правильно обрабатываются события:
  - `onInitialInstall` - создание ярлыков при первой установке
  - `onAppUpdate` - обновление ярлыков при обновлении
  - `onAppUninstall` - удаление ярлыков при удалении
- ✅ Добавлено логирование событий

#### 3. `Contract2512.csproj`
- ✅ Пакет `Clowd.Squirrel` уже был добавлен (версия 2.11.1)
- ✅ Настроен с `PrivateAssets=none` для доступности во временных WPF проектах

### Как это работает:
```csharp
// Прямое использование Squirrel API
using (var updateManager = new UpdateManager(_updateUrl))
{
    var updateInfo = await updateManager.CheckForUpdate();
    
    if (updateInfo?.ReleasesToApply?.Count > 0)
    {
        await updateManager.DownloadReleases(updateInfo.ReleasesToApply, 
            p => progress?.Report(p));
        await updateManager.ApplyReleases(updateInfo);
        return true;
    }
}
```
