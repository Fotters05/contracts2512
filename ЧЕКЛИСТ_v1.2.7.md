1. Проверить, что в `Contract2512.csproj` есть строка: `<CopyLocalLockFileAssemblies>true</CopyLocalLockFileAssemblies>`
2. Выполнить: `dotnet clean`
3. Выполнить: `dotnet publish Contract2512.csproj -c Release -r win-x64 --self-contained false`
4. Проверить папку `bin\Release\net8.0-windows\win-x64\publish\` - DLL должна быть там
5. Если DLL есть локально, но отсутствует в релизе - проблема в GitHub Actions workflow

### Проблема: Ошибка при запуске приложения
**Решение:**
1. Проверить логи в папке: `%LocalAppData%\Contract2512\`
2. Проверить, что все необходимые DLL присутствуют в папке `app-1.2.7\`
3. Попробовать переустановить приложение

### Проблема: Автообновление не работает
**Решение:**
1. Убедиться, что Clowd.Squirrel.dll присутствует

3. Проверить, что на GitHub есть более новая версия (например, v1.2.8)
