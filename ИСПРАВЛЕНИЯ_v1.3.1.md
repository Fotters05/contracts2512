# 🔧 Исправления в версии 1.3.1

## 🎯 Главное исправление: Улучшен поиск Clowd.Squirrel.dll

### ❌ Проблема в v1.3.0:
GitHub Actions workflow упал с ошибкой:
```
❌ Clowd.Squirrel.dll НЕ найден в NuGet кэше!
Error: Process completed with exit code 1.
```

**Причина**: Скрипт искал DLL только в одном фиксированном пути:
```powershell
$squirrelDll = Get-ChildItem -Path "$env:USERPROFILE\.nuget\packages\clowd.squirrel\2.11.1\lib\net8.0\Clowd.Squirrel.dll"
```

На GitHub Actions runner (Windows) переменная `$env:USERPROFILE` может указывать на другого пользователя (например, `runneradmin`), или пакеты могут быть в глобальном кэше.

### ✅ Решение в v1.3.1:
Улучшен скрипт поиска DLL - теперь проверяет **несколько возможных путей** с рекурсивным поиском.

#### Изменения в `.github/workflows/build-and-release.yml`:

**БЫЛО (v1.3.0)**:
```yaml
- name: Copy Clowd.Squirrel.dll manually
  run: |
    $squirrelDll = Get-ChildItem -Path "$env:USERPROFILE\.nuget\packages\clowd.squirrel\2.11.1\lib\net8.0\Clowd.Squirrel.dll" -ErrorAction SilentlyContinue
    if ($squirrelDll) {
      Copy-Item $squirrelDll.FullName -Destination "bin/Release/net8.0-windows/win-x64/publish/" -Force
    } else {
      Write-Host "❌ Clowd.Squirrel.dll НЕ найден в NuGet кэше!"
      exit 1
    }
```

**СТАЛО (v1.3.1)**:
```yaml
- name: Copy Clowd.Squirrel.dll manually
  run: |
    Write-Host "🔍 Поиск Clowd.Squirrel.dll..." -ForegroundColor Cyan
    
    # Проверяем несколько возможных путей
    $searchPaths = @(
      "$env:USERPROFILE\.nuget\packages",
      "$env:ProgramData\NuGet\packages",
      "C:\Users\runneradmin\.nuget\packages"
    )
    
    $squirrelDll = $null
    
    # Сначала проверяем стандартные пути
    $nugetCache = "$env:USERPROFILE\.nuget\packages\clowd.squirrel\2.11.1\lib\net8.0\Clowd.Squirrel.dll"
    $globalCache = "$env:ProgramData\NuGet\packages\clowd.squirrel\2.11.1\lib\net8.0\Clowd.Squirrel.dll"
    
    if (Test-Path $nugetCache) {
      $squirrelDll = $nugetCache
      Write-Host "✅ Найден в стандартном кэше"
    } elseif (Test-Path $globalCache) {
      $squirrelDll = $globalCache
      Write-Host "✅ Найден в глобальном кэше"
    } else {
      # Ищем рекурсивно
      foreach ($searchPath in $searchPaths) {
        if (Test-Path $searchPath) {
          $found = Get-ChildItem -Path $searchPath -Filter "Clowd.Squirrel.dll" -Recurse -ErrorAction SilentlyContinue | 
                   Where-Object { $_.FullName -like "*\2.11.1\lib\net8.0\*" } | 
                   Select-Object -First 1
          if ($found) {
            $squirrelDll = $found.FullName
            Write-Host "✅ Найден через поиск: $squirrelDll"
            break
          }
        }
      }
    }
    
    if (-not $squirrelDll) {
      Write-Host "❌ Clowd.Squirrel.dll НЕ найден!"
      Write-Host "Проверенные пути:"
      foreach ($path in $searchPaths) {
        Write-Host "  - $path"
      }
      exit 1
    }
    
    # Копируем найденный файл
    Copy-Item $squirrelDll -Destination "bin/Release/net8.0-windows/win-x64/publish/" -Force
    Write-Host "✅ Clowd.Squirrel.dll скопирован"
```

#### Как это работает:
1. **Быстрая проверка** стандартных путей (без рекурсии)
2. **Рекурсивный поиск** в нескольких возможных NuGet кэшах
3. **Подробная диагностика** - выводим где нашли DLL или какие пути проверили
4. **Копирование** найденного файла в publish папку
5. **Проверка** что файл успешно скопирован

### 🎯 Ожидаемый результат:
- ✅ Workflow успешно находит Clowd.Squirrel.dll независимо от того, где он находится
- ✅ DLL копируется в publish папку
- ✅ Squirrel.exe упаковывает DLL в установщик
- ✅ После установки приложение запускается без ошибки "Could not load file or assembly"

## 📝 Технические детали:

### Возможные пути NuGet кэша на GitHub Actions:
1. **Пользовательский кэш**: `C:\Users\runneradmin\.nuget\packages\`
2. **Глобальный кэш**: `C:\ProgramData\NuGet\packages\`
3. **Переменная окружения**: `$env:USERPROFILE\.nuget\packages\`

### Почему v1.3.0 не сработал:
На GitHub Actions runner переменная `$env:USERPROFILE` может указывать на:
- `C:\Users\runneradmin` (пользователь runner)
- Но пакеты могут быть в глобальном кэше `C:\ProgramData\NuGet\packages`

Поэтому нужно проверять **все возможные пути**.
