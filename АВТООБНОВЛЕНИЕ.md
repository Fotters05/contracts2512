# 🚀 Автообновление приложения

## Как это работает

Приложение использует **Clowd.Squirrel** для автоматического обновления через GitHub Releases.

### Процесс обновления:

1. **При запуске** приложение проверяет наличие новой версии на GitHub
2. **Если найдено обновление** - показывается окно с предложением обновиться
3. **Пользователь нажимает "Обновить"** - обновление скачивается и устанавливается
4. **Приложение перезапускается** с новой версией

---

## 📝 Как выпустить обновление

### Шаг 1: Измени версию в проекте

Открой `Contract2512.csproj` и увеличь версию:

```xml
<Version>1.2.1</Version>
<AssemblyVersion>1.2.1.0</AssemblyVersion>
<FileVersion>1.2.1.0</FileVersion>
```

### Шаг 2: Создай тег и запуш на GitHub

```powershell
git add .
git commit -m "Release v1.2.1"
git tag v1.2.1
git push origin main
git push origin v1.2.1
```

### Шаг 3: Готово!

GitHub Actions автоматически:
- Соберет проект
- Создаст Setup.exe и пакеты обновления
- Опубликует Release

## 🔍 Как проверить, что все работает

### 1. Проверь текущую версию

Открой `Contract2512.csproj` и посмотри версию (например, `1.2.0`)

### 2. Создай новую версию

```powershell
# Измени версию в Contract2512.csproj на 1.2.1
git add Contract2512.csproj
git commit -m "Bump version to 1.2.1"
git tag v1.2.1
git push origin main
git push origin v1.2.1
```

### 3. Подожди 5-10 минут

GitHub Actions создаст Release с файлами:
- `Contract2512Setup.exe` - установщик
- `RELEASES` - файл версий
- `Contract2512-1.2.1-full.nupkg` - полный пакет
- `Contract2512-1.2.1-delta.nupkg` - дельта-обновление

### 4. Запусти старую версию приложения

Должно появиться окно с предложением обновиться!

## ❓ Частые проблемы

### Обновление не появляется?

**Проверь:**
1. ✅ Тег создан и запушен: `git push origin v1.2.1`
2. ✅ GitHub Actions завершился успешно (зеленая галочка)
3. ✅ Release создан и содержит файлы
4. ✅ Версия в Release новее текущей

### Как посмотреть логи?

Логи автообновления выводятся в Debug консоль Visual Studio.

### Где хранятся обновления?

Приложение устанавливается в:
```
C:\Users\[Username]\AppData\Local\Contract2512\
```

## 🎯 Важно!

- **Обновление работает ТОЛЬКО для установленного приложения** (через Setup.exe)
- **Из Visual Studio обновление НЕ работает** (это нормально)
- **Тег должен начинаться с `v`** (например, `v1.2.1`)
- **Версия в проекте должна совпадать с тегом** (без `v`)
