# 🚀 Как выпустить обновление приложения

## ⚠️ ВАЖНО: Обычный `git push` НЕ создает обновление!

GitHub Actions запускается **ТОЛЬКО при создании тега** (v1.0.0, v1.1.0 и т.д.)

---

## 🎯 Быстрый способ (рекомендуется)

### Используй скрипт `release.ps1`:

```powershell
# Создать релиз версии 1.1.5
.\release.ps1 -Version 1.1.5

# С кастомным сообщением
.\release.ps1 -Version 1.1.5 -Message "Исправлен парсер ДОП"
```

**Скрипт автоматически:**
- ✅ Обновит версию в `Contract2512.csproj`
- ✅ Создаст commit
- ✅ Создаст тег `v1.1.5`
- ✅ Запушит на GitHub
- ✅ GitHub Actions создаст Release с обновлением

---

## 📝 Ручной способ

### 1. Обнови версию в `Contract2512.csproj`

```xml
<PropertyGroup>
  <Version>1.1.5</Version>
  <AssemblyVersion>1.1.5.0</AssemblyVersion>
  <FileVersion>1.1.5.0</FileVersion>
</PropertyGroup>
```

### 2. Создай commit и тег

```bash
# Commit изменений
git add .
git commit -m "Release v1.1.5"

# Создать тег
git tag v1.1.5

# Запушить код и тег
git push origin main
git push origin v1.1.5
```

### 3. Готово!

GitHub Actions автоматически:
- Соберет проект
- Создаст Setup.exe и пакеты обновления
- Опубликует Release на GitHub

---

## 🔍 Проверка

### 1. Проверь GitHub Actions
https://github.com/Fotters05/contracts2512/actions

Должен появиться процесс "Build and Release" (займет ~5-10 минут)

### 2. Проверь Release
https://github.com/Fotters05/contracts2512/releases

После завершения сборки появится новый Release с файлами:
- `Setup.exe` - установщик
- `RELEASES` - файл версий
- `Contract2512-1.1.5-full.nupkg` - полный пакет
- `Contract2512-1.1.5-delta.nupkg` - дельта-обновление

### 3. Проверь обновление в приложении

Запусти старую версию приложения → должно появиться окно обновления!

## ❓ Частые вопросы

### Почему обновление не появляется?

**Проверь:**
1. ✅ Тег создан и запушен на GitHub (`git push origin v1.1.5`)
2. ✅ GitHub Actions завершился успешно (зеленая галочка)
3. ✅ Release создан и содержит файлы `Setup.exe`, `RELEASES`, `.nupkg`
4. ✅ Версия в Release новее, чем в приложении

### Как удалить неправильный тег?

```bash
# Локально
git tag -d v1.1.5

# На GitHub
git push origin --delete v1.1.5
```

### Можно ли пропустить версию?

Да! Можно сразу выпустить v1.2.0, пропустив v1.1.5

### Что делать, если GitHub Actions упал с ошибкой?

1. Проверь логи в Actions
2. Исправь ошибку
3. Удали тег (см. выше)
4. Создай тег заново
