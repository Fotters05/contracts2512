# 📋 ИСПРАВЛЕНИЯ В ВЕРСИИ 1.2.8

## 🎯 ГЛАВНОЕ ИСПРАВЛЕНИЕ

### Проблема в v1.2.7
В версии 1.2.7 была попытка исправить отсутствие `Clowd.Squirrel.dll` через добавление:
```xml
<CopyLocalLockFileAssemblies>true</CopyLocalLockFileAssemblies>
```

**НО** это не сработало, потому что GitHub Actions workflow использовал:
```bash
dotnet publish --self-contained false
```

При `--self-contained false` .NET не включает все зависимости, даже с `CopyLocalLockFileAssemblies=true`.

### Решение в v1.2.8
Изменен `.github/workflows/build-and-release.yml`:

```yaml
# БЫЛО (v1.2.7 и ранее):
- name: Build and Publish
  run: dotnet publish Contract2512.csproj -c Release -r win-x64 --self-contained false -p:PublishSingleFile=false

# СТАЛО (v1.2.8):
- name: Build and Publish
  run: dotnet publish Contract2512.csproj -c Release -r win-x64 --self-contained true -p:PublishSingleFile=false
```

## 📝 ТЕХНИЧЕСКИЕ ДЕТАЛИ

### Что делает `--self-contained true`:
1. ✅ Включает .NET Runtime в публикацию
2. ✅ Включает ВСЕ NuGet зависимости (включая Clowd.Squirrel.dll)
3. ✅ Приложение может работать на машинах без установленного .NET 8
4. ⚠️ Увеличивает размер установщика (~60-80 MB вместо ~20 MB)

### Что делает `--self-contained false`:
1. ❌ НЕ включает .NET Runtime
2. ❌ НЕ включает некоторые NuGet зависимости
3. ❌ Требует установленный .NET 8 на целевой машине
4. ✅ Меньший размер установщика (~20 MB)

### Почему v1.2.7 не сработала:
- `CopyLocalLockFileAssemblies=true` работает только для локальной сборки
- При `--self-contained false` dotnet publish игнорирует некоторые зависимости
- Clowd.Squirrel.dll не была включена в публикацию

## 🔧 ИЗМЕНЕНИЯ В ФАЙЛАХ

### Альтернативные решения (если v1.2.8 не сработает):
1. Использовать `PublishTrimmed=false` для отключения обрезки
2. Явно указать `<IncludeNativeLibrariesForSelfExtract>true</IncludeNativeLibrariesForSelfExtract>`
3. Использовать custom MSBuild targets для копирования DLL
4. Переключиться на другую систему обновлений (Velopack, ClickOnce)

## ⚠️ ВАЖНЫЕ ЗАМЕЧАНИЯ
