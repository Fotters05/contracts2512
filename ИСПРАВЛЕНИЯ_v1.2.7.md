# 🔧 Исправления в версии 1.2.7

## ❌ КРИТИЧЕСКАЯ ПРОБЛЕМА
**Clowd.Squirrel.dll отсутствовала в опубликованном приложении**

### Симптомы:
- При запуске установленного приложения v1.2.4, v1.2.5, v1.2.6 возникала ошибка:
  ```
  Could not load file or assembly 'Clowd.Squirrel, Culture=neutral, PublicKeyToken=null'
  Не удается найти указанный файл.
  ```
- Система автообновления не могла работать, так как DLL отсутствовала

### Причина:
- В `Contract2512.csproj` не было настройки `CopyLocalLockFileAssemblies`
- При публикации с `--self-contained false` NuGet зависимости не копировались в выходную папку
- Squirrel упаковывал папку без Clowd.Squirrel.dll
- Установленное приложение не могло загрузить отсутствующую сборку

### ✅ Решение:
Добавлена настройка в `Contract2512.csproj`:
```xml
<CopyLocalLockFileAssemblies>true</CopyLocalLockFileAssemblies>
```

Эта настройка **заставляет dotnet publish копировать ВСЕ NuGet зависимости** в выходную папку, включая:
- Clowd.Squirrel.dll
- Все остальные библиотеки из NuGet пакетов

## 📝 Изменения в файлах:

### Contract2512.csproj
- ✅ Версия изменена: `1.2.6` → `1.2.7`
- ✅ Добавлена настройка: `<CopyLocalLockFileAssemblies>true</CopyLocalLockFileAssemblies>`
- ✅ Добавлен комментарий: `<!-- CRITICAL: Force copy all NuGet dependencies to output (including Clowd.Squirrel.dll) -->`
